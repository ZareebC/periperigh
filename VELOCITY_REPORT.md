# Peri Peri Grill House — Comprehensive GEO Website Rebuild Report
### Velocity Group | April 2026

## Executive Summary

We rebuilt periperigh.com from scratch — replacing a GoDaddy Website Builder site that scored **48/100** on AI visibility with a purpose-built, AI-optimized Next.js application. The old site gave AI crawlers almost nothing to work with: ~50 words of content, zero structured data, no schema markup, no FAQ, no menu with prices, and no location-specific pages. The new site has **3,300 lines of code across 41 source files**, 10 fully-built pages, 6 JSON-LD schema types, embedded geo-coordinates, AI referrer tracking, and a complete `llms.txt` file — all server-side rendered so AI crawlers get clean HTML on first request.

## Audit Score Breakdown — What We Fixed

The original audit scored the site **48/100**. Here's what each category looked like and what we addressed:

### Website AI-Readiness: 3/15 → Fixed
**Problem:** The GoDaddy site was essentially invisible to AI. JavaScript-heavy rendering meant crawlers got empty pages. No structured data, no schema, no meaningful text content.

**What we built:**
- Server-side rendered pages — every page delivers complete HTML on first request, no JavaScript execution needed
- JSON-LD structured data on every page (6 different schema types)
- Rich text content across all 10 pages (~5,000+ total words)
- Proper semantic HTML: unique H1 per page, H2/H3 hierarchy, descriptive alt text
- Meta descriptions on every page

### Content Structure: 2/15 → Fixed
**Problem:** The old site had no H1 tags, multiple duplicate H2s, and almost no readable content for AI to parse or quote.

**What we built:**
- 10 pages with unique, descriptive H1 tags
- Proper heading hierarchy (H1 → H2 → H3)
- Menu page with full item names, descriptions, and prices (49 items)
- FAQ page with 10 Q&A pairs (also in FAQ schema)
- About page with 800+ words on peri peri history, cooking process, halal commitment
- Hicksville suburb landing page with 500+ words targeting local queries
- Blog infrastructure ready for future content

### AI Visibility: 10/30 → Addressed
**Problem:** The restaurant showed up in AI results but always ranked behind Frangos and Peri-Peri Guys because those competitors had richer content signals.

**What we built to change this:**
- Complete `llms.txt` file — when AI crawlers visit the site, they get a perfectly structured summary of the business, menu, locations, hours, and press coverage
- Press coverage from NYT, Gothamist, and Newsday now referenced in schema `sameAs`, on-page, and in `llms.txt` — this directly competes with Frangos' Infatuation coverage
- Restaurant schema with `servesCuisine: ["Peri Peri", "Halal", "African", "Portuguese", "Chicken"]` — AI now knows exactly what this restaurant serves
- Menu schema with every item and price — when someone asks "how much is peri peri chicken at Peri Peri Grill House," the answer is in the structured data

### Google Business Profile: 13/15 → No change (separate from website)
**Recommendation:** Client should add Velocity as a GBP Manager to optimize descriptions, Q&A, categories, and attributes.

### Reviews & Reputation: 10/15 → Partially addressed
**Problem:** Mixed review signals hurting AI reputation.

**What we built:** FAQ answers addressing common concerns, rich "about" content establishing quality and commitment. Full fix requires active review management (outside website scope).

### AI Bot Access: 10/10 → Maintained
**Already good:** The old site allowed all crawlers. The new site maintains this and explicitly names AI-specific bots in robots.txt.

## What We Built — Technical Inventory

### Architecture
- **Framework:** Next.js 16 with App Router
- **Rendering:** Server-side rendered (SSR) — critical for AI crawlers
- **Hosting:** Vercel (free tier, global CDN, auto SSL)
- **Styling:** Tailwind CSS with custom design system ("Ember & Char" aesthetic)
- **Typography:** Syne (display) + Source Serif 4 (body) + JetBrains Mono (labels)
- **Contact form:** API route with Resend email integration
